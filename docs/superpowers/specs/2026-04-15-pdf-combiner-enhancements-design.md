# PDF Combiner Enhancements — Design Spec

**Date:** 2026-04-15
**Status:** Approved

---

## Goal

Improve the combined supporting-documents PDF with: a grouped index page (by agenda item), PDF bookmarks for quick navigation, a per-document cover page, a footer crediting the tool, a City of Mitcham heading, and a clickable link to the full agenda.

---

## Background

`pdf_combiner.py` currently produces a flat table-of-contents cover page and merges all PDFs. The `parent_item_text` (agenda item descriptor, e.g. "Library Opening Hours") is already extracted by `aidocs_html_analyzer_sorted.py` and present in `target_downloads`, but is not currently passed through to `combine_pdfs`.

---

## Changes

### 1. `scheduled_run.py` — pass `item_descriptor` and `meeting_url`

When building `pdf_entries`, add `item_descriptor` from `parent_item_text`:

```python
pdf_entries = [
    {
        'filepath': os.path.join(result['meeting_folder'], t['recommended_filename']),
        'item_number': t.get('parent_item_number', ''),
        'item_descriptor': t.get('parent_item_text', ''),
        'title': t.get('title', ''),
    }
    for t in result.get('target_downloads', [])
]
```

Pass `meeting_url=url` to `combine_pdfs`.

---

### 2. `pdf_combiner.py` — rewrite

#### `combine_pdfs` signature

```python
def combine_pdfs(pdf_entries, output_path, meeting_title, run_date=None, meeting_url=None):
```

- `meeting_url`: optional string, the Player.aspx URL for the meeting. Displayed as a clickable link on the index page. If `None`, the link row is omitted.
- Each entry in `pdf_entries` gains an optional `item_descriptor` key (the agenda item title, e.g. "Library Opening Hours").

#### Index page layout

Grouped by agenda item (Option B from brainstorming):

```
City of Mitcham — [meeting_title]
[date]  ·  Supporting Documents
Full agenda: [meeting_url as clickable link]
────────────────────────────────────────────
10.1  Draft Policy Review                          (bold heading + underline)
    Supporting document — Rates Hardship Policy    p. 2
10.4  Library Opening Hours
    Supporting document — Library Opening Hours Schedule 2026    p. 28
    Supporting document — Slides — Community Consultation Results    p. 35
────────────────────────────────────────────
Generated by Mitcham Council Supporting Documents Scraper  ·  2026-01-27
```

Grouping logic:
- Group entries by `item_number` + `item_descriptor`, preserving order of first occurrence.
- Within a group, list each document with its page number (right-aligned).
- If `item_descriptor` is empty, show only the item number as the group heading.
- Page overflow: if the table runs past the bottom margin, start a new page and continue.

Document title displayed on index: strip the `"Supporting Document \d+\s*[-:]?\s*"` prefix (existing logic), then prepend `"Supporting document — "`.

#### Per-document cover page (Option A — minimal)

A single page inserted immediately before each supporting document's pages:

```
[small grey, top]
City of Mitcham  ·  [meeting_title]  ·  [run_date]

[medium bold, centre]
Item [item_number]  ·  [item_descriptor]

[large bold, centre]
Supporting document — [display_title]

[small grey, bottom rule]
Supporting document N of M for this item
```

- `N of M`: count within the item group (e.g. "1 of 2").
- If `item_descriptor` is empty, the middle line shows only `Item [item_number]`.
- Cover page is included in the page count for bookmarks (bookmark points to the cover page).

#### PDF bookmarks

After all pages are assembled, add a nested PDF outline:

```
10.1  Draft Policy Review
  └─ Supporting document — Rates Hardship Policy           → page N
10.4  Library Opening Hours
  └─ Supporting document — Library Opening Hours Schedule 2026    → page N
  └─ Supporting document — Slides — Community Consultation Results → page N
```

- Top-level bookmark: `[item_number]  [item_descriptor]` (or just `[item_number]` if no descriptor). Points to the first cover page of the first document in the group.
- Child bookmark: `Supporting document — [display_title]`. Points to that document's cover page.
- Implemented with `pypdf`'s `add_outline_item` / `add_bookmark` API.

#### Clickable agenda link on index page

Use `reportlab`'s `canvas.linkURL(url, rect)` to make the URL text on the index page a clickable hyperlink in the PDF.

### 3. `tests/test_pdf_combiner.py` — update existing tests

Update the `combine_pdfs` calls to pass `item_descriptor` in entries (existing tests pass empty string — no behaviour change). Add new tests:

- Index page output contains "City of Mitcham"
- Index page output contains the meeting URL when `meeting_url` is provided
- Index page omits the URL row when `meeting_url=None`
- Per-document cover pages are inserted (page count = N docs × (1 cover + doc pages) + 1 index)
- PDF outline has correct number of top-level bookmarks
- Grouped index: two entries with same `item_number` appear under one heading
- `item_descriptor` empty: group heading shows item number only
